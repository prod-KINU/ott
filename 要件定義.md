2. design.mdとの整合・競合一覧

* 0px radius
* Electric Blueをインタラクションに限定

* full-widthで視認性の高い上部ナビ
* Graphite系フッター

⸻

2-2. 競合候補

競合1: ハンバーガーメニュー画像演出

事実
一部画像使用は許容されている。
競合
画像演出が強いと、design.md の機能主義・静けさとぶつかる。
推奨整理
画像はトリガー周辺または補助装飾に限定し、リンク本体は文字主体にする。

競合2: loading画面

事実
3枚画像切替は確定。
競合
演出が濃いとミニマルな世界観と衝突する。
推奨整理
フェード最小、装飾最小、音なし、フルスクリーンで静かに切替。

競合3: homeのGIF運用

事実
将来差し替え可能にする。
競合
GIF主体だとノイズ感や可読性低下の恐れ。
推奨整理
初期実装は静止画優先、GIFは差し替え可能なオプションに留める。

競合4: JP / EN切替UI

事実
単一ページ内切替。
競合
トグルや装飾を強くするとアクセントが過多になる。
推奨整理
JP / EN のテキストリンク型を採用し、active状態のみ明示。

6. マイクロインタラクション / ローディング演出の提案

基本方針

* 派手にしない
* 速度感は機敏
* hover / focus / active だけを丁寧に作る
* reduced motion 対応あり

推奨する演出

1. ナビリンク

* hoverで Electric Blue に変化
* 下線またはボーダーの微細変化
* durationは短め

2. projectsグリッド

* hoverで border / text / image の軽い変化
* scaleは極小か不使用
* active item は border色または小さなインジケータで強調

3. モバイルメニュー

* スライドインは可
* ただし速度は速く、背景オーバーレイは薄く
* 装飾的なイージングは避ける

4. mobile project detail

* オーバーレイというよりページ遷移風
* 横スライドまたはフェード＋位置補正を極小で

5. loading

* 3枚画像の切替
* クロスフェードは最小限
* テキストがあるとしても極小
* 読み込み終了時は即座に消える

6. reduced motion

* 画像切替以外のトランジションは大幅簡略化
* 不要なtransformは停止
